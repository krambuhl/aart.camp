## Contract

- **Goal**: Add a draft-owned project resolver mirroring loom's
  `resolveProject` behavior, but selecting for trout-style projects.
  This is the seam draft needs because loom's resolver excludes draft's
  projects by design (loom filters its `listProjects` to directories
  containing `manifest.json`, which trout projects don't have). Draft's
  `revise` and `read` verbs (D5, D6) will use this resolver to find
  existing projects on disk. Test-first via vitest; mirrors the loom
  resolver tests' shape against a temp-dir fixture.

- **Acceptance criteria**:
  - New file `.claude/cli/lib/draft-project.ts` exporting:
    ```ts
    export function resolveTroutProject(slugOrPath: string, projectsRoot: string): string
    ```
  - Behavior parallels loom's `resolveProject`:
    - Absolute or relative path ‚Üí resolve and check existence; throw
      `project-not-found` if missing.
    - Full slug (matches `SLUG_RE`) ‚Üí look up in active list first,
      then archived; throw `project-not-found` if absent.
    - Date-less suffix ‚Üí suffix-match across active; if 1 match
      return; if >1 throw `slug-ambiguous` with candidates; if 0 fall
      to archived list with same rules.
  - **Trout filter**: a directory at `projects/<slug>/` counts as a
    trout project iff it contains `MANIFEST.md` (case-sensitive) AND
    does NOT contain `manifest.json`. Both conditions required ‚Äî
    excludes loom projects AND non-substrate directories.
  - Companion `listTroutProjects(projectsRoot, opts?)` exported
    alongside (mirrors loom's `listProjects` interface). Used
    internally; exported so future code can introspect.
  - Reuses `LoomError`, `SLUG_RE`, `DATELESS_RE`, `ARCHIVE_DIRNAME`
    from `./project.ts`. The slug-grammar primitives stay in loom's
    lib; only the resolver differs.
  - New file `.claude/cli/lib/draft-project.test.ts` with vitest
    coverage parallel to `project.test.ts`:
    - happy: full slug ‚Üí path
    - happy: date-less suffix ‚Üí unique match
    - happy: archived slug falls back to `archive/`
    - happy: relative path resolves to absolute
    - error: nonexistent slug throws `project-not-found`
    - error: ambiguous suffix throws `slug-ambiguous` with candidates
    - filter: loom-only project (`manifest.json` present) does NOT
      resolve via `resolveTroutProject`
    - filter: directory without `MANIFEST.md` does NOT resolve
    - `listTroutProjects`: returns only trout projects (filters out
      loom + non-substrate directories)
    - `listTroutProjects`: `--archived` option enumerates archive
  - All existing tests still pass.
  - `npm run lint`, `npm run test`, `npm run build` clean.

- **Disqualifiers**:
  - Defining the function in `draft.ts` or anywhere outside
    `.claude/cli/lib/`.
  - Duplicating slug-grammar constants (`SLUG_RE`, `DATELESS_RE`,
    `ARCHIVE_DIRNAME`) ‚Äî import from `./project.ts`.
  - Skipping the trout/loom filter test cases (these are the
    coexistence boundary; the whole point of the file).
  - Implementation-first (must commit failing tests first).

## Notes for the PR

- This is the first draft-cli file that lives outside the shared
  primitives layer. It's the trout/loom boundary in code ‚Äî draft
  owns its own resolver because loom's now filters out trout
  projects.
- Slug-grammar regexes (`SLUG_RE`, `DATELESS_RE`) are duplicated
  here rather than imported because `./project.ts` does not export
  them. The file's leading comment documents the choice. If a future
  PR exports those constants from `./project.ts`, this file can
  import them and drop the local duplicates. Worth a follow-up
  ticket; not in scope here.
- The trout filter is two-sided by design: requires `MANIFEST.md`
  AND requires the absence of `manifest.json`. The absence check
  guards against a project transitioning from trout to loom (both
  files present temporarily) accidentally resolving from either
  side. The transition itself is out of scope, but the resolver is
  resilient to it.
- Pre-existing test failure noted in Verification. Not a regression
  caused by this checkin; substrate-side cleanup needed separately.
- D3 unblocks D5 (`revise` verb) and D6 (`read` verb), both of
  which use `resolveTroutProject` to find existing projects.
