**Unit**: D2 ‚Äî Author `.claude/cli/draft.ts` entry + `bin/draft` shim,

## Contract

- **Goal**: Stand up the draft CLI entry point and bash shim, with
  identical structure to loom-cli's (parseInvocation ‚Üí dispatch ‚Üí
  main, pure helpers exported for direct testing, side-effecting
  `main()` only at the bottom). Three verbs are registered in a flat
  `VERBS` map; each dispatches to a `not-implemented` placeholder so
  D3/D4/D5 can plug in actual verb handlers without restructuring.
  Tests parallel loom.test.ts in shape: pure-helper coverage plus
  subprocess smoke tests via `spawnSync` against both `node .../draft.ts`
  and `bin/draft`.

- **Acceptance criteria**:
  - `.claude/cli/draft.ts` exists with `#!/usr/bin/env node` shebang.
  - Exports a flat `VERBS: Record<string, string>` registry mapping
    verb name ‚Üí one-line purpose. Three entries: `plan`, `revise`,
    `read`.
  - Exports `parseInvocation(argv: string[]): Invocation` where
    `Invocation = {kind: 'help'} | {kind: 'unknown', verb: string} | {kind: 'verb', verb: string, rest: string[]}`.
    Behavior:
    - empty argv ‚Üí help
    - `--help` or `-h` anywhere in argv ‚Üí help
    - first arg starts with `-` or empty ‚Üí unknown
    - first arg matches a key in VERBS ‚Üí verb (with rest)
    - otherwise ‚Üí unknown
  - Exports `formatHelp(): string` listing all VERBS entries with their
    purposes; mentions JSON-by-default convention; mentions
    `LOOM-CONVENTIONS.md` cross-reference for substrate conventions.
  - Exports `formatUnknownVerbError(verb: string): string` emitting a
    JSON string with `{error: 'unknown-verb', message, candidates}`
    where `candidates` is `Object.keys(VERBS)`.
  - Exports `DispatchResult = {stdout?, stderr?, exitCode}` and
    `dispatch(invocation: Invocation): DispatchResult`. Returns
    help ‚Üí stdout + exit 0; unknown ‚Üí stderr + exit 1; verb (no impl
    yet) ‚Üí stderr + exit 1 with `{error: 'not-implemented', message,
    verb}` payload.
  - `main(argv: string[]): never` invokes parseInvocation ‚Üí dispatch
    ‚Üí writes streams + exits. Uses the `isEntryPoint()` guard same as
    loom.ts so the file is importable without running.
  - `bin/draft` exists, is executable (`chmod +x`), and is a bash
    shim that `exec node` invokes `.claude/cli/draft.ts` with passed
    args. Mirror `bin/loom` structure exactly (same set -euo
    pipefail, same SHIM_DIR/REPO_ROOT resolution, same
    `loom-shim-error:` style error message but with `draft-shim-error:`
    prefix).
  - `.claude/cli/draft.test.ts` exists with the following test
    coverage (mirrors loom.test.ts):
    - parseInvocation: no args ‚Üí help
    - parseInvocation: `--help` ‚Üí help
    - parseInvocation: `-h` ‚Üí help
    - parseInvocation: known verb (e.g. `plan`) ‚Üí verb with rest
    - parseInvocation: unknown verb ‚Üí unknown
    - parseInvocation: `--help` after verb still routes to help
    - formatHelp lists every entry in VERBS
    - formatUnknownVerbError emits structured JSON with candidates
    - dispatch: help ‚Üí stdout + exit 0
    - dispatch: unknown ‚Üí stderr + exit 1
    - dispatch: known verb (no impl yet) ‚Üí not-implemented error
    - node entry --help via spawnSync prints help, exits 0
    - node entry no-args via spawnSync prints help, exits 0
    - node entry unknown verb via spawnSync prints structured error,
      exits 1
    - `bin/draft` shim via spawnSync `--help` prints help, exits 0
  - All existing tests in the project still pass.
  - `npm run lint`, `npm run test`, `npm run build` clean.

- **Disqualifiers**:
  - Implementing any verb body in this checkin ‚Äî all three must stub

## Notes for the PR

- D2 ships the CLI scaffold only; verb handlers come in D3-D5.
  Each verb stubs to a `not-implemented` payload with a forward
  pointer to the next deliverable.
- Mirrors loom.ts / bin/loom / loom.test.ts as close to 1:1 as
  possible: pure-helper structure, isEntryPoint guard, parseArgs
  forward-compat with `--pretty`. Names, padding, and error prefixes
  are the only intentional divergences. Future readers should be
  able to read both side-by-side and spot the (flat vs
  namespaced) difference instantly.
- The decision to ship `bin/draft` as a near-verbatim copy of
  `bin/loom` (rather than refactor to a shared shim helper) is
  intentional: two CLIs, two shims, no premature abstraction. If
  a third CLI ever appears, extract.
- `parseArgs` is called for top-level flags only here; verb-level
  argument parsing (`--plan-file=`, `--rationale=`, etc.) lives in
  the verb handlers when D3-D5 land.
