# Checkin 01 — ev.draft-cli.plan-verb

**Created**: 2026-05-15
**Phase**: 1 — Ship CLI + two skills + loop wiring
**Unit**: D4 — Implement the `plan` verb. Pure verb function + thin
IO wrapper. Test-first. Creates `projects/<date>-<slug>/`, writes
PLAN.md and INTERVIEW.md from `--plan-file` / `--interview-file`,
optionally `git add` + `git commit`. Refuses on committed PLAN.md;
allows overwrite on uncommitted PLAN.md (recovery from failed commit).

## Contract

- **Goal**: Author the first real draft CLI verb. `bin/draft plan
  <slug-or-topic> --plan-file=<path> --interview-file=<path>
  [--no-commit] [--pretty]` creates a new trout-style project
  directory and commits PLAN.md + INTERVIEW.md from the provided
  files. The verb is the load-bearing primitive `/draft-plan` (D7
  skill) will shell out to. D4 ships the verb in
  `.claude/cli/verbs/draft.ts` (the draft equivalent of loom's
  `verbs/project.ts`) with one entry in a `DRAFT_VERBS` registry,
  wires it into the entry-file dispatch in `.claude/cli/draft.ts`,
  and lands a small git wrapper in `.claude/cli/lib/draft-git.ts`
  that `revise` (D5) will share.

- **Acceptance criteria**:
  - New file `.claude/cli/verbs/draft.ts` exports:
    - `DraftCliContext` type with `{ projectsRoot, today?, gitRunner? }`.
    - `DispatchResult` type (re-uses loom's shape).
    - `planVerb(rest: string[], ctx: DraftCliContext): DispatchResult`.
    - `DRAFT_VERBS: Record<string, VerbHandler>` with `plan: planVerb`
      (revise and read added in D5 / D6).
  - New file `.claude/cli/lib/draft-git.ts` exports:
    - `GitRunner` type with two methods:
      ```ts
      isCommitted(repoRoot: string, filePath: string): boolean
      addAndCommit(repoRoot: string, paths: string[], message: string): void
      ```
    - `defaultGitRunner` — real implementation shelling out to `git`
      via `child_process.spawnSync`.
  - `planVerb` behavior, in order:
    1. Parse args via `parseArgs`. Required positional:
       `slug-or-topic`. Required flags: `--plan-file`,
       `--interview-file`. Optional: `--no-commit`, `--pretty`.
       Missing required → `LoomError('missing-args', ...)` →
       structured stderr + exit 1.
    2. Resolve target slug:
       - If `slug-or-topic` matches `SLUG_RE` (already a full date-
         prefixed slug) AND a trout project at that slug exists →
         `LoomError('project-already-exists', ...)`.
       - Otherwise → treat as topic, derive `slug =
         createSlug(topic, today)`. Use `ctx.today` if provided
         (test injection) else `new Date().toISOString().slice(0,10)`.
       - If the derived slug resolves to an existing trout project →
         `LoomError('project-already-exists', ...)`.
    3. Compute `targetDir = join(projectsRoot, slug)`.
    4. Collision check on `targetDir/PLAN.md`:
       - If file exists AND is committed (`ctx.gitRunner.isCommitted`)
         → `LoomError('plan-exists-committed', ...)` with message
         suggesting `revise`.
       - If file exists but uncommitted → log "overwriting
         uncommitted PLAN.md" to stdout (advisory) and proceed.
    5. `mkdirSync(targetDir, { recursive: true })`.
    6. `copyFileSync(planFile, join(targetDir, 'PLAN.md'))`.
    7. `copyFileSync(interviewFile, join(targetDir,
       'INTERVIEW.md'))`.
    8. Unless `--no-commit`: `ctx.gitRunner.addAndCommit(repoRoot,
       [planMd, interviewMd], "[draft plan] <slug>")`.
    9. Return `{ stdout: JSON.stringify({slug, path: targetDir,
       committed: !noCommit}), exitCode: 0 }`. `--pretty` produces
       indented JSON.
  - Entry file wiring:
    - `.claude/cli/draft.ts` imports `DRAFT_VERBS` and
      `DraftCliContext` from `./verbs/draft.ts`.
    - `dispatch(invocation, ctx)` signature updated: when
      `invocation.kind === 'verb'`, look up the handler in
      `DRAFT_VERBS[invocation.verb]`; if present, call it; if
      absent (revise/read), keep returning `not-implemented` (so
      D5/D6 can wire in later without restructuring).
    - `main()` constructs a `DraftCliContext` and passes it to
      `dispatch`.
  - New `.claude/cli/verbs/draft.test.ts` with vitest coverage:
    - **Happy path**: `planVerb` with valid args writes both files,
      calls `addAndCommit`, returns the expected JSON envelope.
    - **--no-commit**: writes files but does NOT call
      `addAndCommit`. Result JSON has `committed: false`.
    - **dir-exists-no-PLAN**: target directory exists but PLAN.md
      doesn't → succeeds (creates files).
    - **uncommitted-overwrite**: PLAN.md exists but git's
      `isCommitted` returns false → overwrites without error.
    - **plan-exists-committed**: PLAN.md exists AND
      `isCommitted` returns true → throws
      `plan-exists-committed`.
    - **project-already-exists**: slug-or-topic matches an existing
      trout project on disk → throws `project-already-exists`.
    - **missing required args**: no positional → throws
      `missing-args`. No `--plan-file` → throws `missing-args`. No
      `--interview-file` → throws `missing-args`.
    - **slug derivation**: passing a topic-shaped string derives
      slug from `createSlug(topic, ctx.today)`; the returned
      `path` reflects that slug.
  - Update `.claude/cli/draft.test.ts`:
    - The existing test `dispatch: known verb (no impl yet) →
      not-implemented error` currently uses `verb: 'plan'`. Change
      it to `verb: 'revise'` (still a stub) so the not-implemented
      shape is still exercised; plan now has an impl.
    - Add a new test: `dispatch: 'plan' verb routes to planVerb`
      using a stubbed handler in DRAFT_VERBS (or simply assert
      that calling dispatch with verb='plan' and missing args
      returns a `missing-args` error, not a `not-implemented`
      error).
  - All existing tests still pass.
  - `npm run lint`, `npm run test`, `npm run build` clean.

- **Disqualifiers**:
  - Skipping the pure/effect split — `planVerb` must be testable
    against an injected `gitRunner` (tests don't actually call
    `git`).
  - Hard-coding today's date — must be injectable via
    `ctx.today` for deterministic test output.
  - Skipping the uncommitted-overwrite case (it's the recovery
    path; explicitly called out in PLAN's Risks).
  - Implementing revise or read in this checkin (those are D5 /
    D6).
  - Putting the git wrapper in `verbs/draft.ts` instead of `lib/draft-
    git.ts` (revise needs it too).

- **Inputs**:
  - `.claude/cli/loom.ts` — pattern for entry-file dispatch
    composition
  - `.claude/cli/verbs/project.ts` — pattern for verb-namespace
    file shape (especially `projectScaffold` — closest analog)
  - `.claude/cli/lib/project.ts` — `createSlug`,
    `LoomError`
  - `.claude/cli/lib/draft-project.ts` — `resolveTroutProject` to
    detect already-existing projects on disk
  - `.claude/cli/draft.ts` — existing stub-dispatching entry to be
    updated
  - `projects/2026-05-15-draft-cli/PLAN.md` § CLI surface, §
    Decisions (pure/effect split, --no-commit, collision rules)

**Approach**:
- Authored the git wrapper first since it defines the test
  injection seam (`GitRunner`).
- Test-first per contract. 13 vitest cases against the not-yet-
  existent `verbs/draft.ts`. Stubbed `GitRunner` captures method
  calls and tracks a `committedPaths: Set<string>` for the
  collision tests.
- Implemented `planVerb` with explicit pure/effect ordering: parse
  → validate → resolve slug → resolve collision → mkdir/copy →
  optionally commit → emit JSON.
- The slug-resolution branch is two-stage: if `slugOrTopic` matches
  `SLUG_RE` (already date-prefixed), use as-is; else derive via
  `createSlug`. Either way, check via `resolveTroutProject` whether
  the slug already names a trout project; refuse if so.
- Wired `DRAFT_VERBS` into entry dispatch. The `notImplemented`
  helper inside `verbs/draft.ts` keeps the not-implemented JSON
  shape consistent for revise/read until D5/D6 fill them in.
- Smoke test via bin/draft against /tmp/draft-smoke: creates
  `2026-05-15-test-topic/PLAN.md` + INTERVIEW.md, returns correct
  JSON envelope.

## Notes for the PR

- The git wrapper (`lib/draft-git.ts`) lands here even though it's
  consumed by both D4 and D5 (revise). Putting it in this PR rather
  than a standalone deliverable keeps D5's PR small and focused on
  the verb logic.
- `DraftCliContext` is intentionally distinct from loom's
  `CliContext` — loom carries a `ghRunner` (PR/GitHub work) where
  draft carries a `gitRunner` (local commits). They're parallel
  shapes but the injectable concerns differ.
- The `notImplemented` helper for revise/read keeps a consistent
  JSON shape so downstream callers (and `/draft-plan` skill in D7)
  don't have to special-case the partial-rollout window between
  D4 and D5/D6.
- Smoke test uses `DRAFT_PROJECTS_ROOT` env var override
  (mirroring loom's `LOOM_PROJECTS_ROOT`). Documented in entry
  file by the `deriveProjectsRoot` function.
- `revise` and `read` will each land in their own PRs (D5, D6).
  D5 reuses `lib/draft-git.ts` for the commit step and
  `resolveTroutProject` for finding the existing project. D6 just
  reads.
