## Execution

The wet run drove the production `/griot-compact` skill against
four notes from the unprocessed corpus, picked to exercise
different outcome paths. The skill is `user-invocable` with
`disable-model-invocation: true` so the orchestrator-side calls
were manually staged through the chat session rather than auto-
fired; subagent dispatch and helper-script invocations followed
the SKILL.md prose verbatim.

**Note 1 — `2026-04-30T20-30-03-stack-default-alignment-text`** →
**DID_NOT_REPRODUCE.** Rubric-author produced a clean 3-assertion
rubric; control and treatment outputs were structurally identical
(both proposed `alignment={{ xs: 'start', sm: 'center' }}`).
Round 1 panel split 3 UNCHANGED + 1 DID_NOT_REPRODUCE; round 2
flipped to 3 DID_NOT_REPRODUCE + 1 UNCHANGED (3/4 supermajority).
This is also the note that surfaced the **first rule ambiguity**:
the trailing paragraph in `griot-judge.md` said both "if control
and treatment are effectively identical, the verdict is
UNCHANGED — not DID_NOT_REPRODUCE" *and* "only use DID_NOT_
REPRODUCE when control already passes every assertion." Both
conditions hold simultaneously when control passes everything
and treatment is identical to control. Different judges resolved
it different ways. Rule fix #1 restructured into a precedence-
ordered list with explicit first-match-wins semantics, with
DID_NOT_REPRODUCE taking precedence over UNCHANGED when both
apply (rationale: downstream actions differ — DID_NOT_REPRODUCE
archives, UNCHANGED triggers rewrites; spending rewrite attempts
on a learning that has nothing to improve on is wasted work).

**Note 2 — `2026-05-06T10-22-36-node-24-strips-types`** →

that the control might anchor on existing `tsx` patterns).
Control recommended `node` for substrate but *also* endorsed
`tsx` for root-level scripts ("Leave that alone"); treatment

3 IMPROVED + 1 UNCHANGED on the scope of assertion 3; round 2

**Note 3 — `2026-05-08T14-09-51-regex-no-z-anchor`** → panel

## Notes for PR

- **What this checkin proves and what it does not.** The new
  `/griot-compact` pipeline is end-to-end verified across every
  flow that matters for promotion (rubric-author → control +
  treatment → round 1 → round 2 → mediate → outcome → rollup +
  archive + bench-history) and for the rewriter loop (verify-
  rubric → rewriter → revised `learning.md` on disk). Three paths
  remain unexercised by this wet run: the operator stuck-
  learning escalation (only fires after five failed attempts on
  one note — out of scope), the regression suite §3 (now
  possible because rollup.md has entries; will run automatically
  on the next `/griot-compact` invocation), and the no-consensus
  fallthrough in round 2 (requires a genuinely deadlocked panel
  the wet run did not produce). Phase 3's deletion deliverable
  is gated on the rewriter being proven, which it now is; the
  other paths can be exercised by the first scheduled nightly
  run.

- **Rule fix #1: precedence-ordered list.** The previous rule
  prose had two clauses in tension. The new prose is a four-
  rule first-match-wins list. The reordering matters for the
  case where control passes every assertion and treatment is
  identical: under the old prose, judges split between
  UNCHANGED ("treatment is identical to control") and
  DID_NOT_REPRODUCE ("control passes everything"); under the
  new prose, DID_NOT_REPRODUCE wins because rule 2 fires
  before rule 4. Downstream consequence: notes where the
  failure mode doesn't reproduce go straight to archive
  instead of wasting up to five rewriter attempts on a
  learning that has nothing to improve on.

- **Rule fix #2: drop the strict IMPROVED conjunct.** The
  previous IMPROVED rule required *every* assertion control
  failed to now pass in treatment. Sonnet surfaced that this
  leaves a verdict gap when treatment fixes some but not all
  of control's failures: IMPROVED's conjunction fails, but
  UNCHANGED's "same pass count" also fails (2 ≠ 1). The fix
  drops the second conjunct; rule 1 already catches treatment
  introducing new failures earlier in the precedence chain,
  so "treatment passes strictly more" is sufficient for
  IMPROVED. Partial progress on a multi-assertion rubric
  now correctly classifies as IMPROVED. The fix landed
  *after* the dogfood note ran, so the rewriter trigger
  remains a real wet-run artifact and not an artifact of
  the rule change.

- **The wet-run corpus has a structural bias toward DID_NOT_
  REPRODUCE.** When the corrected artifact lives on disk and
  the prompt invites Claude to read it, the control output
  routinely cribs from the fix and passes every rubric
  assertion. This isn't a pipeline bug — it's an artifact
  of how session-notes are captured (after the fix lands,
  in the same repo). The two notes that hit IMPROVED were
  ones where the learning's failure mode required either
  knowledge of an external-to-repo capability (Node 24
  strip-types) or process-shaped behavior (raising a scope
  fork) — failure modes that reading the codebase doesn't
  resolve. Worth keeping in mind when interpreting the
  first scheduled nightly run's rates; expect a high
  DID_NOT_REPRODUCE share on the existing 22-note backlog.

- **Substantive observation about debate dynamics.** Both
  IMPROVED notes (L-001 and L-002) and the UNCHANGED note
  saw real position shifts between round 1 and round 2. The
  debate-summarizer-then-re-judge design isn't theater —
  judges actually move when they see other reasoning.
  Round 2 in the dogfood note flipped from 3-1 IMPROVED to
  4-0 UNCHANGED, a complete consensus inversion. This is
  evidence the round 2 path is doing real work and shouldn't
  be optimized away (e.g., by collapsing into a single
  round when round 1 is "close enough").

- **Phase 3 deletion is next.** With the new path proven,
  the deletion deliverable (drop `scripts/learnings-compact/`,
  remove the `learnings:compact` npm script, drop API-key
  prereq language from any remaining doc) is now safe to
  ship. Targeted as the next checkin on this branch.
