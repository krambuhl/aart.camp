## Contract

- **Goal**: Add `.claude/scripts/griot/mediate-panel.ts` — a
  deterministic Node script that takes a round of judge subagent
  outputs and returns a structured `PanelRound` with parsed
  verdicts, tally, threshold check, tier-split detection, and
  tiebreak application. Plus a co-located test file
  `mediate-panel.test.ts`. Mirrors the existing
  `.claude/scripts/guild/parse-and-aggregate.ts` shape: stdin
  JSON in, stdout JSON out, invoked via `spawnSync('node',
  [SCRIPT])` from tests. The script absorbs the deterministic
  logic from `scripts/learnings-compact/mediator.ts` (tally,
  threshold check, tier-split detection, tiebreak rule) and
  adds the verdict-text parsing that the SDK formerly got from
  the Anthropic tool-use schema. Phase 2 SKILL.md will pipe each
  round's subagent outputs through this script and act on the
  result.

- **Acceptance criteria**:
  1. `.claude/scripts/griot/mediate-panel.ts` exists. ESM module
     (matches `.claude/scripts/package.json` `"type": "module"`),
     `#!/usr/bin/env node` shebang, runs as `.ts` directly under
     Node 24 (which strips types). Reads JSON from stdin, writes
     JSON to stdout, errors to stderr with prefix
     `mediate-panel-error: `, exits non-zero on any input/parse
     failure that isn't a per-judge parse failure (those become
     errored verdicts in the output, not script-level errors).
  2. Input shape (parsed from stdin):
     ```
     {
       "round_num": 1 | 2,
       "verdicts": [
         { "judge_id": <string>, "tier": <"opus"|"sonnet"|"haiku">, "raw_output": <string> },
         ...
       ],
       "config": {
         "consensus": { "round_1_blind": <int>, "round_2_debate": <int> },
         "tiebreak": { "rule": <string>, "top_tier": <string> }
       }
     }
     ```
     Empty `verdicts`, missing top-level keys, or non-integer
     thresholds → script-level error.
  3. For each input verdict, the script extracts the fenced
     `verdict` block from `raw_output` (the contract from
     `griot-judge.md`), parses the JSON inside, and produces a
     `ParsedVerdict`:
     ```
     { judge_id, tier, verdict, control_evals, treatment_evals,
       reasoning, errored, error_message? }
     ```
     where `verdict` is one of `IMPROVED | UNCHANGED | REGRESSED |
     DID_NOT_REPRODUCE`. If the block is missing or
     unparseable, or `verdict` is not one of those four, mark
     `errored: true` and set `error_message` and `verdict:
     "UNCHANGED"` (placeholder; excluded from tally).
  4. `tally` counts non-errored verdicts:
     `{ IMPROVED, UNCHANGED, REGRESSED, DID_NOT_REPRODUCE }`,
     each defaulting to 0.
  5. `consensus_verdict` is the verdict with the largest tally
     count, but only when `threshold_met` is true. Otherwise
     `null`.
  6. `threshold_met` is `true` iff the largest tally count is
     `>= config.consensus.round_<N>_blind` (round 1) or
     `>= config.consensus.round_<N>_debate` (round 2). Errored
     verdicts do not contribute to counts but are not subtracted
     from the panel size — a panel of 4 with 1 errored cannot
     reach 4/4 unanimity.
  7. `tier_split` is `true` iff every non-errored judge in
     `top_tier` voted one verdict, every non-errored judge NOT in
     `top_tier` voted a single different verdict, and both
     groups are non-empty. Pure detection — never affects
     consensus or tiebreak.
  8. Tiebreak application: only attempted when `round_num == 2`
     AND `threshold_met == false` AND every non-errored judge in
     `top_tier` voted the same verdict AND there are at least
     2 non-errored judges in `top_tier`. When all conditions
     hold, `tiebreak_applied: true` and `tiebreak_verdict` is
     the top-tier's shared verdict. Otherwise
     `tiebreak_applied: false` and `tiebreak_verdict: null`.
     The supported `rule` value is `"top_tier_consensus"`; an
     unknown rule string makes tiebreak a no-op (false / null,
     no error). Tiebreak does NOT change `consensus_verdict` or
     `threshold_met` — those reflect the panel's own decision.
     The orchestrator decides whether to use the tiebreak verdict
     when consensus is null.
  9. Output shape on stdout:
     ```
     {
       "round": <1|2>,
       "verdicts": ParsedVerdict[],
       "tally": Tally,
       "consensus_verdict": Verdict | null,
       "threshold_met": boolean,
       "tier_split": boolean,
       "tiebreak_applied": boolean,
       "tiebreak_verdict": Verdict | null
     }
     ```
     Single-line JSON acceptable; pretty-printed acceptable. Any
     other content on stdout is a violation.
  10. `.claude/scripts/griot/mediate-panel.test.ts` exists and
      uses vitest + `spawnSync('node', [SCRIPT])` to exercise the
      script via stdin/stdout. Tests cover, at minimum:
      - empty stdin → script-level error
      - non-JSON stdin → script-level error
      - missing top-level keys → script-level error
      - round 1 unanimous (4/4 IMPROVED) → consensus IMPROVED,
        threshold_met true
      - round 1 not unanimous (3/4) → consensus null,
        threshold_met false
      - round 2 supermajority (3/4) → consensus, threshold_met
        true, no tiebreak
      - round 2 split 2-2 with both Opus agreeing → tiebreak
        fires, tiebreak_verdict matches Opus
      - round 2 split 2-2 with Opus disagreeing → tiebreak
        does not fire
      - tier_split detection (2 Opus IMPROVED vs 1 Sonnet + 1
        Haiku REGRESSED) → tier_split true
      - errored verdict (malformed verdict block in raw_output)
        → ParsedVerdict has errored: true, error_message set,
        excluded from tally
      - unknown tiebreak rule string → tiebreak no-op (no error)

## Execution

`mediate-panel.ts` is a pure stdin → stdout JSON transformer. No
filesystem mutation, no network, no LLM calls. The script:

1. Reads stdin and parses as JSON. Empty stdin or invalid JSON →
   script-level error on stderr, exit 1.
2. Validates the input shape (`round_num` ∈ {1, 2}; `verdicts` is
   non-empty array of `{judge_id, tier, raw_output}`; `config`
   has `consensus.round_1_blind`/`round_2_debate` integers and
   `tiebreak.rule`/`top_tier` strings). Schema violations → script
   error.
3. For each input verdict, extracts the fenced ` ```verdict `
   block from `raw_output` via regex
   (`/```\s*verdict\s*\n([\s\S]*?)```/`), parses the JSON inside,
   and validates the `verdict` field against the four-value
   enum. Failure at any step → `errored: true` with a specific
   `error_message`; the script does not abort.
4. Tallies non-errored verdicts by verdict type.
5. `pickMajority` iterates the four verdict types in fixed order,
   returning the type with the largest count. Ties resolve to the
   first-iterated type, but that doesn't matter — when there's no
   real majority, the threshold check fails and `consensus_verdict`
   becomes null anyway.
6. `threshold_met` = `majority count >= round threshold`. Errored
   judges count toward neither count nor threshold — a 4-judge
   panel with 1 errored cannot reach 4/4 unanimity.
7. `tier_split` detection: every non-errored top-tier judge voted
   one verdict, every non-errored non-top-tier judge voted a
   different single verdict, both groups non-empty.
8. Tiebreak: only attempted when `round_num === 2` AND
   `!threshold_met`. The supported rule is `top_tier_consensus` —
   needs ≥ 2 non-errored top-tier judges who all agree. Unknown
   rule strings → no-op (false / null), no error.
9. Writes the `Output` JSON to stdout, single trailing newline.

Test file uses the established `spawnSync('node', [SCRIPT])`
pattern from `.claude/scripts/guild/parse-and-aggregate.test.ts`.
17 tests cover all 11 required cases plus extras: malformed
verdict block JSON, unknown verdict value, parsed evals
preservation, non-integer threshold validation, tier-split when
non-top tier disagrees among themselves.

## Notes for PR

- This is the deterministic counterpart to `griot-judge.md` — the
  subagent emits a fenced verdict block; this script parses,
  tallies, decides. Together they replace the SDK's tool-use-based
  judge panel.
- `parseSingleVerdict` returns an `errored` ParsedVerdict on any
  parse failure rather than throwing. The orchestrator's response
  to errored judges is policy (retry? escalate? proceed with
  remaining?) — the script doesn't bake that in.
- Errored judges are counted as "missing" not "discounted." A
  4-judge panel with 1 errored cannot reach 4/4 unanimity. This
  prevents the failure mode where unreliable subagents
  auto-approve their own absence.
- Tiebreak's `rule` field is a string for forward-compatibility.
  Today the only supported rule is `top_tier_consensus`. An
  unknown rule is a no-op (not a hard error) so the orchestrator
  can choose to log and proceed rather than abort the round on a
  config typo.
- Tier-split detection runs unconditionally and is pure
  reporting — it never affects `consensus_verdict` or
  `tiebreak_verdict`. The orchestrator can use it for human-review
  flags without it interfering with the consensus path.
- Phase 2's SKILL.md will pipe each round's subagent outputs into
  this script and read the result. The script's pure-function
  nature lets us unit-test the consensus logic without LLM mocks.
- Next deliverable (D4): `operator-checks.ts` — the parallel
  helper for operator invariants (rubric tampering detection,
  attempt counting, JSONL logging). Last D in Phase 1.
