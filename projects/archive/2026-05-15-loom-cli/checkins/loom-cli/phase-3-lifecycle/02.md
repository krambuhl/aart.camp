# Checkin 02 — loom-cli/phase-3-lifecycle

**Created**: 2026-05-15 10:28
**Phase**: 3 — Lifecycle write API + lifecycle skills
**Unit**: State-update verbs (phase update + checkin/session/retro write)

## Contract

- **Goal**: Wire four state-update verbs into the CLI by composing
  unit 01's writers with auto-event-appending. Each verb performs
  one atomic state change (manifest mutation, or new immutable
  file) plus the corresponding event line in events.jsonl. With
  this unit merged, loom can mutate live project state — phases
  transition, checkins are recorded, sessions are saved, retros
  are written.

- **Acceptance criteria**:
  1. `verbs/phase.ts` gets `phaseUpdate(rest, ctx)` and PHASE_VERBS
     map grows. Args: `<slug> <N> --status=<v> [--branch=<b>]
     [--pr=<n>] [--reason=<text>]`. Reads manifest, finds phase N,
     updates fields, writes manifest, emits one of `phase-started`
     / `phase-completed` / `phase-blocked` / `phase-unblocked`
     based on the transition (see below). Returns the updated
     phase JSON.
  2. Transition → event mapping:
     - status=in-progress, prior=blocked → `phase-unblocked`
     - status=in-progress, prior=anything else → `phase-started`
     - status=completed → `phase-completed`
     - status=blocked → `phase-blocked` (requires `--reason=` flag;
       error if absent)
     - status=not-started → no event (rare; only useful during
       scaffolding/reset)
  3. `verbs/checkin.ts` gets `checkinWrite(rest, ctx)` and
     CHECKIN_VERBS grows. Args: `<slug> --checkin-file=<path>`.
     Reads checkin JSON from file, validates `schema_version === 1`
     and presence of required fields, writes via writeCheckin,
     appends `checkin-created` event. Returns `{number, branch, path}`.
  4. `verbs/session.ts` gets `sessionWrite(rest, ctx)` and
     SESSION_VERBS grows. Args: `<slug> --session-file=<path>`.
     Same pattern: read file, validate, write, append session-saved
     event. Returns `{filename, path}`.
  5. `verbs/retro.ts` gets `retroWrite(rest, ctx)` and RETRO_VERBS
     grows. Args: `<slug> --retro-file=<path>`. Read file, validate,
     write via writeRetro, append `retro-written` event with the
     retro's type (and optional phase/tier for session retros).
     Returns `{filename, path}`.
  6. All four verbs accept `--pretty` for output formatting.
  7. Each verb has tests added to its sibling test file covering
     happy paths + at least one error case per verb (missing args,
     invalid file, blocked-without-reason).
  8. Events written by these verbs are observable via the existing
     `events read` / `events latest` verbs — round-trip test:
     `phase update` then `events latest --event=phase-completed`
     returns the just-emitted event.
  9. TDD discipline: tests-first commit, then impls commit.
  10. `npm run test` passes; biome lint clean.
  11. Verbs intentionally don't read mid-write state — if the
      caller's file is partially-formed JSON, validation fails
      with a structured error rather than writing garbage.

  - Allows phase status=blocked without --reason.

- **TDD step 2** (`887b78f`): 4 verb impls + dispatch map updates.
  All 456 tests green.

Each verb follows the same skeleton: parseArgs for flags, validate
inputs, resolve project path, read+mutate (or read file), write via
unit-01's writer, append event via appendEvent.

`phaseUpdate`'s `eventForTransition` helper is the meaty piece — it
orders the blocked→in-progress branch before the broader in-progress
fallthrough so phase-unblocked is emitted for the right transition.
Status=blocked requires --reason; status=not-started emits no event
(scaffolding-only transition).

Validation in the write verbs is intentionally shallow: schema_version
and the small set of required typed fields. Per PLAN.md, no runtime
JSON Schema. Bad input throws `invalid-checkin` / `invalid-session`
/ `invalid-retro` with the underlying parse error.

## Notes for the PR

- **Loom can now mutate state.** Four write verbs let humans and
  loops modify a loom project's phases, write checkins, save
  sessions, and write retros — each with the appropriate event
  appended to events.jsonl.
- **phaseUpdate's PR-URL placeholder.** When `--pr=N` is provided,
  the phase row gets `pr.url = https://github.com/example/example/pull/N`.
  A future `--pr-url=` flag (or auto-detection from repo origin)
  would replace this; for now it's a tracked placeholder.
- **Shallow validation is by design.** Per PLAN.md, no runtime
  JSON Schema. Write verbs check schema_version + required fields
  + branch/number/date/letter type, throw structured errors on
  failure. Loops authoring files should validate their own output
  before calling.
