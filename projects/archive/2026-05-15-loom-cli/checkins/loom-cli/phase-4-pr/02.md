**Unit**: pr open + pr update

## Contract

- **Goal**: Wire the two PR-write verbs that compose `gh` for
  create/edit operations. Each emits the appropriate event
  (`pr-opened`, `pr-updated`) and updates the manifest phase row's
  PR field when applicable. Both validate inputs and forward gh
  errors as structured LoomError payloads.

- **Acceptance criteria**:
  1. `prOpen(rest, ctx)`: Args `<slug> --title=<t> --body-file=<path>
     [--branch=<b>]`. Runs `gh pr create --title <t> --body-file <p>
     [--head <b>]`. Parses gh stdout (PR URL) to extract the new
     PR number. Appends `pr-opened` event with `{pr, url}` detail.
     Returns `{pr, url}` JSON.
  2. `prUpdate(rest, ctx)`: Args `<slug> --pr=<N> --body-file=<path>`.
     Runs `gh pr edit <N> --body-file <p>`. Appends `pr-updated`
     event with `{pr}` detail. Returns `{pr}` JSON.
  3. Both verbs forward gh non-zero exits as `gh-failed` LoomError
     with the underlying stderr message.
  4. Both validate required flags before touching gh; missing
     args → `missing-args` error.
  5. PR_VERBS map grows `open` and `update` keys.
  6. Sibling tests in `verbs/pr.test.ts` use stubbed ghRunner.
     Tests assert: happy paths produce events, body-file content
     is forwarded to gh, missing args reject before gh runs, gh
     failure surfaces as gh-failed.
  7. TDD discipline.
  8. `npm run test` passes; biome lint clean.

- **Rules applied**:
  - TDD.
  - Stdlib only.
  - Erasable TS.
  - Compose `runGh` (or injected ghRunner) for all gh invocations.
  - LoomError shape.
  - No emojis.

- **Disqualifiers**:
  - Implements pr comments/respond (unit 03).
  - Implements loom-pr / loom-pr-respond skills (unit 04).
  - Hardcodes gh path.
  - Lets the PR URL parsing depend on a specific repo host (must
    handle the standard `https://github.com/<owner>/<repo>/pull/<N>`
    shape gh emits).

  - `.claude/cli/lib/events.ts` — appendEvent

prOpen composes gh + URL parse + event append. PR_URL_RE matches

forks, GitHub Enterprise, etc. prUpdate is simpler — gh edit, then
event append.

exit, network errors, auth required) surface as `gh-failed`
LoomError with the underlying error message preserved.

## Notes for the PR

- **gh-failure handling.** Both verbs wrap the gh call in try/catch
  and surface failures as structured `gh-failed` errors. The
  underlying error message is preserved in the LoomError detail.
- **Host-agnostic URL regex.** `PR_URL_RE` matches any
  `https://*/pull/N` URL. Works against github.com, GitHub
  Enterprise hosts, forks — anything gh writes to stdout that
  looks like a pull-request URL.
- **No manifest phase-row update yet.** prOpen/prUpdate emit events
  but don't update the manifest's phase.pr field (that integration
  lives in the eventual `loom-pr` skill or as a follow-up to
  `phase update --pr=N`). Keeping the gh-side and manifest-side
  separate makes the verbs composable.
