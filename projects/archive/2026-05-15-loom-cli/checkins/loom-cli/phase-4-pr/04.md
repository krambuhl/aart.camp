## Contract

- **Goal**: Author the final two `loom-*` skills that complete the
  loom system. `loom-pr` composes a PR title + body from checkin
  JSON and drives `pr discover` + `pr open`/`update` to land the
  PR. `loom-pr-respond` reads comments, classifies them, drafts
  responses, and writes them via `pr respond`. With these
  shipped, Phase 4 closes and the loom-cli project is complete.

- **Acceptance criteria**:
  1. `.claude/skills/loom-pr/SKILL.md` exists with proper
     frontmatter (`name: loom-pr`, scoped allowed-tools). Body
     documents the four-state PR workflow: invoke `loom pr discover`,
     dispatch on state, compose title + body (single-checkin and
     multi-checkin shapes), invoke `loom pr open` or
     `loom pr update`. Refuses to act on `drift` state.
  2. PR body composition includes the marker
     `<!-- loom-pr-checkins: NN[,NN,...] -->` so subsequent
     `pr discover` calls can detect freshness.
  3. Title format: single-checkin → `[Phase N] <unit name>`;
     multi-checkin (all same phase) → `[Phase N] <phase name>`
     (from manifest's phase row).
  4. `.claude/skills/loom-pr-respond/SKILL.md` exists with
     similar frontmatter. Body documents the flow: invoke
     `loom pr comments`, classify each comment (blocker / advisory
     / question / nit / ignore), draft a response per comment,
     compose a responses-file JSON, invoke `loom pr respond`.
  5. Both skills compose `bin/loom` at the seams, never
     `node .claude/cli/loom.ts`.
  6. Both preserve the `correction:` convention (read-only —
     surfacing corrections from upstream checkins where relevant).
  7. Skills are prose; no code; no tests beyond format validity.
  8. `npm run test` unchanged (502/503 — pre-existing main
     breakage; loom-side stays green).
  9. biome lint clean.
  10. Phase 4 closes after this unit; loom-cli project becomes
      archive-eligible.

## Execution

Single commit (`c4d0c00`) authoring two SKILL.md files. Skills are
prose, not code — no TDD split applies.

loom-pr mirrors trout-pull-request's four-state machine but composes
from Checkin JSON (vs markdown sections) and uses the loom-prefixed
`<!-- loom-pr-checkins: -->` marker. Section caps and "motivation
must be sourced, never invented" rule carry over from the trout
substrate's established conventions.

(blocker / advisory / question / nit / ignore) as the LLM-shaped
contribution. The CLI returns comments verbatim; the skill decides
which deserve responses and what to say. Posting back to GitHub is
explicitly downstream — the skill stops at local file writes.

on Write — confirmed visible in the system-reminder list. Test
suite unchanged (502/503; the 503rd is the pre-existing main
