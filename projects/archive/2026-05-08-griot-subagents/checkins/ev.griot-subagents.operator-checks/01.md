     - Writes `{"ok": true, "appended_to": <log_path>}` to
       stdout, exits 0. (The success acknowledgement is JSON so
       the orchestrator can parse it uniformly.)

## Execution

`operator-checks.ts` is a single script with two subcommand modes
dispatched via the first positional argv entry. The mode dispatch
sits at the top of `main()`; each mode handler validates its own
stdin shape and either writes a JSON result to stdout or fails to
stderr.

**Mode dispatch (`readMode`)** — checks `process.argv[2]` against
the `VALID_MODES` tuple. Missing or unknown mode → script-level
error listing the valid modes.

**Stdin parsing (`readJsonStdin`)** — common to both modes. Empty
stdin or invalid JSON → script-level error.

**`verifyRubric`** — validates `{rubric_path, expected}` shape,
checks the rubric file exists (script error if missing — wrong
path is an orchestrator bug), reads it, byte-compares to
`expected`. Result is either `{ok: true}` or `{ok: false, actual:
<full content>}` written to stdout. No file mutation; the function
is pure read-and-compare.

**`logIntervention`** — validates `{log_path, record}` shape (uses
`'record' in obj` rather than typeof since `record` may be any JSON
value including `null`, arrays, scalars). Creates the parent
directory recursively (`mkdirSync({recursive: true})`), then
appends `JSON.stringify(record) + "\n"` to `log_path`. Writes
`{ok: true, appended_to: <log_path>}` acknowledgement on stdout
so the orchestrator can parse the success uniformly with the
verify-rubric path.

Test file uses `mkdtempSync(join(tmpdir(), 'operator-checks-test-
'))` in `beforeEach` for isolated fixtures, `rmSync` in
`afterEach` for cleanup. Nothing leaks into the working tree.

## Notes for PR

- Two-mode dispatch in one script is a small departure from the
  one-job pattern in sibling scripts (`mediate-panel.ts`,
  `parse-and-aggregate.ts`). The grouping is intentional — both
  operations exist to support the operator role's invariant work
  and naturally cluster. PLAN.md named the deliverable singular
  (`operator-checks.ts`) on this premise.
- `verify-rubric` errors when the rubric file is missing rather
  than returning `{ok: false, actual: ""}`. A missing file is an
  orchestrator bug (wrong path), not a tampering signal — failing
  loudly surfaces the bug. Tampering produces `ok: false` with
  the actual content; that's the orchestrator's signal to log an
  intervention and abort.
- `log-intervention` accepts any JSON value as `record` (object,
  array, primitive, null) — schema is owned by the orchestrator,
  not this script. Tests verify all five JSON kinds round-trip.
- The acknowledgement on stdout (`{ok: true, appended_to: ...}`)
  for `log-intervention` matches the shape of `verify-rubric`'s
  output. The orchestrator can parse uniformly.
