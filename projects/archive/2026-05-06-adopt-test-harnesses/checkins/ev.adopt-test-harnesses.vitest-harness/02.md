- **Goal**: Write a small example test against `wrapResponsive` to prove the
  vitest harness actually runs real assertions on real source. End state:
  `npm test` collects and passes the test, the `--passWithNoTests` crutch

  (lint, build, test) is green.

- **Acceptance criteria**:
  - Test file exists at `utilities/opaque-responsive.test.ts` (co-located
    with source — modern vitest convention, no new directory invented)
  - The test imports `wrapResponsive` and `unwrapResponsive` via the `@/*`
    path alias (proves alias resolution works end-to-end)
  - The test covers 2–3 representative cases of `wrapResponsive` behavior:
    - Undefined input → empty responsive object
    - Single value input → wrapped under `xs`
    - Already-responsive object input → wrapped intact
  - Test uses explicit imports (`import { describe, it, expect } from 'vitest'`)
    — no globals
  - `package.json` `test` script changes from
    `"vitest run --passWithNoTests"` back to `"vitest run"` (the crutch is
    no longer needed)
  - `npm test` → 1 passed, exit 0
  - `npm run build` → exit 0 (still works)
  - `npm run lint` → exit 0 (Biome clean on the new test file)

  - New test infrastructure pulled in (custom matchers, factories, helpers)

## Execution

Created `utilities/opaque-responsive.test.ts` co-located with the source.
Test suite has 3 cases covering the three semantic branches of
`wrapResponsive`:

1. `undefined` input → empty responsive object
2. Single-value input → wrapped under the `xs` breakpoint
3. Already-responsive object input → wrapped intact, preserving keys

Imports use `@/utilities/opaque-responsive` (alias path) — exercises the
`@/*` resolver from D1's `vitest.config.ts` end-to-end. Explicit
`import { describe, expect, it } from 'vitest'` per the no-globals
contract.

Tests round-trip via `unwrapResponsive` for assertion. Any other approach
(reaching into the symbol-keyed internals of `OpaqueResponsive`) would be
fighting the opaque contract.

Updated `package.json`: removed `--passWithNoTests` from the `test`
script. Now that a real test exists, the crutch is no longer needed and
its absence makes "no tests collected" failures (e.g. broken globs)
loud rather than silent.

## Notes for the PR

This PR adopts vitest as the unit-test harness for the project. It is
intentionally scoped as **infrastructure only** — there is one example
test (`utilities/opaque-responsive.test.ts`, 3 cases) whose job is to
prove the harness runs, not to begin a coverage push.

What's added:
- `vitest@^4.1.5` + `@testing-library/react`, `@testing-library/jest-dom`,
  `jsdom` as devDependencies
- `vitest.config.ts` (jsdom env, `@/*` path alias mirroring tsconfig,
  vite's built-in CSS Modules handling)
- `vitest.setup.ts` (jest-dom matchers via vitest's `expect.extend`)
- `npm test` (run-once) and `npm test:watch` scripts
- One test file as a smoke screen

What's deferred:
- Broader unit/component test coverage (Phase 2 introduces Storybook
  and adds the first component test against `Stack`)
- CI wiring (Phase 4)
