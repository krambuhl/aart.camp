## Execution

Two pivots happened at contract negotiation. First, the user
redirected from Path A (Biome-only static signal) to "Path B +
playwright expert" — bundling jsx-a11y, Playwright, and axe-core
into D1. The contract was rewritten and a bounded devDep set
committed: `eslint`, `eslint-plugin-jsx-a11y`,
`@typescript-eslint/parser`, `@axe-core/playwright`. All four
installed; an `eslint.config.mjs` flat config was authored scoped
to jsx-a11y recommended rules over `app/` and `components/`; an
`npm run lint:a11y` script was added; `npm run lint:a11y` was run
and came back clean on the existing JSX.

Second, the user then asked "do I need eslint?" — a direct
pull-back. Honest answer: Biome's `a11y/` recommended group already
covers ~70% of jsx-a11y's recommended rules; the delta is mostly
nice-to-haves; the genuinely additive signal is axe-core at
runtime, not jsx-a11y at static-analysis time. User picked the
simplification. ESLint, `eslint-plugin-jsx-a11y`, and
`@typescript-eslint/parser` were uninstalled. `eslint.config.mjs`
was deleted. The `lint:a11y` script was removed. The contract was
rewritten a third time to reflect a single-devDep bounded set
(`@axe-core/playwright`) and two signals: Biome `a11y/` static
(via existing `npm run lint`) + axe-core runtime (via new
`npm run test:a11y`).

## Notes for the PR

- **Two-signal a11y surface, one new devDep.** Static signal is
  Biome's `a11y/` rule subset (already part of `npm run lint`, no
  new tool). Runtime signal is `@axe-core/playwright` (new devDep)
  wired into a sibling Playwright config so a11y runs don't drag
  along the Storybook build that `test:e2e` requires. Net cost: one
  package, one config file, two test files, two npm-script lines,
  one permission.

- **correction: ESLint considered, then rejected mid-flight.** The
  contract was first written as Biome-only (Path A), then rewritten
  for jsx-a11y + Playwright + axe per user redirect, then rewritten
  again after the user pushed back on the ESLint surface area. The
  rollback removed 3 devDeps and 180 transitive packages. The final
  shape is honest to "Biome is the linter, keep it basic" while
  preserving the axe-core runtime catch-rate that genuinely adds
  signal a static linter can't reach.

- **correction: PLAN's "jsx-a11y" mention is partially superseded.**
  PLAN.md § Phase 2 names "jsx-a11y" parenthetically as the CLI
  validator. The actual implementation uses Biome's `a11y/` rules
  for static + axe-core for runtime instead. The catalog cites
  Biome rule codes (not jsx-a11y rule codes) for tool-detectable
  entries. Worth a one-line update to PLAN.md at phase close, but
  not in this checkin.

- **Sketch test excludes `<canvas>` deliberately.** p5.js generates
  the canvas dynamically; canvas-level a11y (aria-label describing
  the sketch, fallback content) is a separate concern at the
  component-API layer. The exclusion is documented inline so the
  rationale is one click away.

- **The evaluator's "when neither signal applies" branch matters.**
  Substrate edits (agent files, scripts, project docs) have no JSX
  scope and no rendered-page scope. The evaluator returns approved
  with a note rather than firing `packet-incomplete`. This avoids
  noise on every non-frontend unit the panel evaluates.
