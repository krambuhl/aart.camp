3. **Matching algorithm decision**. First-pass implementation
   used last-match-wins (iterate rule list from end). Tests
   exposed a problem: `*.module.css` comes BEFORE `*.css` in
   PANEL-COMPOSITION.md's table, so the LAST match for a
   `.module.css` file was the more-general `*.css` rule —
   dropping the `naming` lens. **correction:** switched to
   most-specific-wins, scored by literal-character count
   (excluding `*` wildcards) with slash count as a secondary
   signal and table position as tiebreaker. This decoupling
   means rule order in the spec is cosmetic — humans can
   rearrange rows for readability without breaking the
   matching. Test cases covering substrate-script-over-`*.ts`,
   substrate-test-over-substrate-script, and `.module.css`-
   over-`.css` validate the change.

4. **Glob anchoring decision**. Patterns without `/` (e.g.
   `*.tsx`, `*.module.css`) are basename patterns — they match
   the FINAL path segment regardless of directory depth, so
   `*.tsx` matches both `Foo.tsx` and `components/Foo.tsx`.
   Patterns containing `/` (e.g. `.claude/agents/*.md`) anchor
   at the start of the path. The two behaviors are documented
   in `globToRegex`'s header comment.
