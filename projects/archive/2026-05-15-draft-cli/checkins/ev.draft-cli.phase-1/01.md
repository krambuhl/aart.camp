## Contract

- **Goal**: Add an exported `createSlug` function to
  `.claude/cli/lib/project.ts` (alongside `resolveProject`,
  `listProjects`, the slug regexes). Returns `<YYYY-MM-DD>-<slugified-
  topic>` matching the existing `SLUG_RE` pattern. Throws `LoomError`
  on invalid inputs. Test-first via vitest, mirroring loom-cli's
  fixture+failing-test+implement discipline. This is the first
  deliverable of phase 1 and a tiny one ‚Äî naming primitive only, no
  CLI surface yet.

- **Acceptance criteria**:
  - New exported function in `.claude/cli/lib/project.ts`:
    ```ts
    export function createSlug(topic: string, today: string): string
    ```
  - Returns a string of the shape `<YYYY-MM-DD>-<slug>` where:
    - `<YYYY-MM-DD>` is the verbatim `today` arg (validated)
    - `<slug>` is `topic` slugified: lowercase; runs of
      non-alphanumeric chars collapsed to a single `-`; leading and
      trailing `-` trimmed; multiple consecutive `-` collapsed to one
  - The returned string matches the existing `SLUG_RE` exactly
    (`/^\d{4}-\d{2}-\d{2}-[a-z0-9][a-z0-9-]*[a-z0-9]$/`).
  - Throws `LoomError` (from `./errors.ts`) on invalid inputs:
    - empty or whitespace-only `topic` ‚Üí `invalid-topic` with message
    - `topic` that slugifies to empty (only special chars, e.g.
      `"---"` or `"!!!"`) ‚Üí `invalid-topic`
    - `today` not matching `/^\d{4}-\d{2}-\d{2}$/` ‚Üí `invalid-date`
  - `.claude/cli/lib/project.test.ts` extended with:
    - happy path: `createSlug("Adopt Biome", "2026-05-15")` ‚Üí
      `"2026-05-15-adopt-biome"`
    - special chars: `createSlug("CLI: plan & revise!", "2026-05-15")`
      ‚Üí `"2026-05-15-cli-plan-revise"`
    - collapses double-spaces / runs:
      `createSlug("hello  world", "2026-05-15")` ‚Üí
      `"2026-05-15-hello-world"`
    - trims leading/trailing punctuation:
      `createSlug("--draft-cli--", "2026-05-15")` ‚Üí
      `"2026-05-15-draft-cli"`
    - empty topic throws `invalid-topic`
    - whitespace-only topic throws `invalid-topic`
    - all-special-chars topic throws `invalid-topic`
    - malformed date throws `invalid-date` (e.g. `"05/15/2026"`,
      `"2026-5-15"`, `""`)
  - All existing tests in `.claude/cli/lib/project.test.ts` still pass.
  - `npm run lint` clean.
  - `npm run test` clean.

- **Disqualifiers**:
  - Defining `createSlug` anywhere other than
    `.claude/cli/lib/project.ts`.
  - Skipping any edge-case test listed above.
  - Breaking any existing test in `project.test.ts`.
  - Implementation-first (no failing test committed before
    implementation).
  - Importing anything beyond what `project.ts` already imports plus
    `LoomError`.

- **Inputs**:

  - `projects/2026-05-15-draft-cli/PLAN.md` ¬ß Decisions ‚Äî

  - `projects/2026-05-15-draft-cli/whiteboards/1-ship-cli-skill-and-loop-wiring.md`
