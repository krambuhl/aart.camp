**Acceptance criteria**:

1. **Script: `.claude/scripts/trout/archive-relocate.ts`.** ESM, stdlib-only (`node:util`, `node:fs`, `node:path`, `node:child_process`). Argument surface: one positional argument resolved like `autosave.ts`'s `resolveProject` (exact slug under `projects/` → suffix match → full path). The script does two things in this order: (a) read `<project>/MANIFEST.md`, find the `**Status**: active` line in the header block, replace `active` with `archived`, write back; (b) `git mv <project> projects/archive/<basename>` via `execFileSync('git', ['mv', ...])`. Stdout on success: one line `relocated: <old-path> → <new-path>`. Stderr on failure: `archive-relocate-error: <reason>` and exit 1.

2. **Preconditions enforced.** The script refuses (and reports) when:
   - The resolved project path doesn't exist (`project not found: <path>`).
   - The project is already under `projects/archive/` (`already archived: <path>`).
   - The MANIFEST.md `**Status**` line isn't `**Status**: active` (`unexpected status: expected "active", found "<value>"; manifest at <path>`). Catches half-finished prior runs and projects in non-active states (e.g. `paused`).
   - The destination `projects/archive/<basename>/` already exists (`destination already exists: <path>`).
   - `git mv` exits non-zero — surface git's stderr verbatim under the `archive-relocate-error: git mv failed: <stderr>` prefix.

3. **Tests: `.claude/scripts/trout/archive-relocate.test.ts`.** Mirrors the pattern in `capture.test.ts` and `autosave.test.ts`: `mkdtempSync` fixtures, `execFileSync('node', [SCRIPT, ...])`, `git init` inside the fixture so `git mv` works. Cover: (a) happy path (slug form) — Status field rewritten, directory moved, stdout matches, working tree shows the rename in `git status`; (b) happy path (path form, both `./projects/<slug>` and absolute); (c) project not found; (d) project already in `projects/archive/`; (e) Status not `active` (test both a literal `paused` value and a missing field); (f) destination already exists; (g) `git mv` failure surfaces git's stderr (induce by removing the `.git/` dir from the fixture before invoking). Tests use `import { test } from 'vitest'` with `// @vitest-environment node` directive at the top of the file (matches the autosave precedent — the deferred test-env decision lands on per-file rather than project-level config).

If git is partially configured (e.g. `.git/` exists but the source path isn't tracked), `git mv` would also fail and the rollback would still fire — but that case isn't separately tested.
