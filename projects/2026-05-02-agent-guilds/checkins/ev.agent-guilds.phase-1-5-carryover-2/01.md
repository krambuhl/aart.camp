## Contract

- **Goal**: Tighten the `commit` verb's untracked-files staging policy in `.claude/scripts/trout/pr-plumbing.ts`. Currently every untracked file outside `projects/` gets staged automatically — which is "too permissive for code outside `projects/`" per session-2026-05-08-b refinement item #2. Replace the implicit allow-everything path with an explicit curated allowlist of code directories + a small set of root-level config files. Untracked paths outside the allowlist are surfaced as a refusal error before any staging happens, so the human can either move them, add a `.gitignore` entry, or extend the allowlist intentionally rather than discovering them after-the-fact in a commit.

- **Acceptance criteria**:
  1. New constants (or equivalent) on `pr-plumbing.ts`: a directory-prefix allowlist for untracked-file staging outside `projects/`, plus a small explicit allowlist for repo-root config files. Initial set, derived from the existing aart.camp layout:
     - **Prefix allowlist**: `.claude/`, `app/`, `components/`, `sketches/`, `styles/`, `lib/`, `tokens/`, `public/`, `scripts/`, `learnings/`
     - **Root files allowlist**: `package.json`, `package-lock.json`, `tsconfig.json`, `biome.json`, `next.config.ts`, `next.config.js`, `postcss.config.mjs`, `vitest.config.ts`, `next-env.d.ts` (modulo `STAGE_EXCLUDES` — `next-env.d.ts` is already excluded; the entry here just makes the allowlist semantically complete)
  2. `selectStagePaths` (or a new helper layered on it) returns both **staged paths** and **refused paths**. A refused path is any untracked file that's not under `projects/<slug>/checkins/<branch>/`, not in the prefix allowlist, and not in the root-files allowlist.
  3. `verbCommit` checks the refused list **before** staging. If non-empty: surface a clear error to stderr listing every refused path and the suggested resolutions (3 options: move under an allowlisted dir; add to `.gitignore`; extend the allowlist if it's a legitimately new code area), then exit non-zero. **No partial staging on refusal** — the whole commit fails closed.
  4. Tracked-modified files are still staged regardless of location (modulo `STAGE_EXCLUDES`). The tightening only applies to **untracked** files. Rationale: a tracked-modified file is already known to git, so it doesn't carry the same "stray-file" risk as untracked.
  5. The existing checkin-file staging path (untracked files under `projects/<slug>/checkins/<branch>/` matching `NN.md`) continues to work unchanged. Other untracked files under `projects/` continue to be silently ignored (existing behavior — they're project-state files outside the current branch's checkins, e.g. PLAN.md edits, MANIFEST.md mutations, sessions/, etc.).
  6. Sibling `pr-plumbing.test.ts` adds coverage for:
     - allowlisted-prefix untracked file (e.g. `.claude/scripts/foo/bar.ts`) → staged successfully
     - root-files-allowlist untracked file (e.g. `package.json` after `npm install` regenerated it) → staged successfully
     - non-allowlisted untracked file outside `projects/` (e.g. `random-scratch.md` at repo root, or `notes/scratch.md`) → refusal error surfaces with the path listed and the 3 resolution suggestions present
     - mixed: one allowlisted + one refused untracked → still refuses (fail-closed semantics; no partial staging)
     - tracked-modified file outside the allowlist → staged successfully (allowlist applies to untracked only)
     - untracked file under `projects/<slug>/checkins/<branch>/NN.md` → staged successfully (existing checkin path)
     - untracked file under `projects/<slug>/PLAN.md` → silently ignored (existing project-state path)
  7. `selectStagePaths` (or its successor) remains pure (exported, deterministic, easy to test directly with synthetic input arrays). Existing pure-helper tests in `pr-plumbing.test.ts` continue to pass.
  8. `npm run lint`, `npm run build`, `npm run test` all clean. Total substrate test count goes up (new tests added; no existing tests removed).
  9. No new runtime dependencies; no behavioral changes beyond the documented tightening.

## Execution

Three additions to `pr-plumbing.ts`, all in the "pure helpers" block above `verbCommit`:

1. Two named exports for the allowlist itself: `UNTRACKED_PREFIX_ALLOWLIST` (array of 10 directory prefixes) and `UNTRACKED_ROOT_FILE_ALLOWLIST` (Set of 9 root-level filenames). The list IS the project-shape spec — discoverable by name, extensible deliberately.
2. New pure helper `isUntrackedAllowlisted(path)`: returns `true` if path starts with any allowlisted prefix OR exact-matches a root-files entry. Used by `selectStagePaths`.
3. `selectStagePaths` refactored to return `StageOutcome = { stage: string[]; refused: string[] }` (was: `string[]`). Untracked files outside `projects/` and outside the allowlist now land in `refused` instead of being silently staged. Tracked-modified files and the existing `projects/<slug>/checkins/<branch>/NN.md` path are unchanged.

`verbCommit` checks `outcome.refused` **before** any staging. Non-empty → emit a multi-line error (`pr-plumbing-error: untracked-paths-refused: <count> untracked paths outside the commit allowlist:` followed by the offending paths and three resolution options: move under allowlisted dir, add to `.gitignore`, or extend the allowlist constant). Exit non-zero. **No partial staging on refusal** — the whole commit fails closed, as specified by AC #3 and the no-partial-staging disqualifier.

Two pre-existing integration tests (`commit: stages substrate paths and pushes by default` and `commit: excludes settings.local.json and next-env.d.ts`) used root-level test files (`someCode.ts`, `normal.ts`) that would now be refused. Both were updated to write the test file under `lib/` (an allowlisted prefix) — the test intent is unchanged, just the path.

## Notes for the PR

- **Fail-closed semantics are the most-reviewer-relevant design choice.** Mixed allowlisted + refused untracked → no partial staging, no commit. The alternative (warn-and-skip) would silently drop the user's intended changes — the kind of substrate violation that compounds invisibly. The integration test at `pr-plumbing.test.ts:678-684` locks in this property by asserting `git log` line count stays at 1 (the fixture's init commit) after a mixed invocation.
- **Allowlist as named exports, not function-local constants.** `UNTRACKED_PREFIX_ALLOWLIST` and `UNTRACKED_ROOT_FILE_ALLOWLIST` are top-of-file exports — the list IS the project-shape spec and deserves to be discoverable. Future projects forking this substrate get a clear extension point. The disqualifier "Hardcoding paths inside `selectStagePaths`" was specifically called out in the contract.
- **The allowlist intentionally doesn't cover all root-level files.** Things like `.gitignore`, `.gitattributes`, `README.md`, `CLAUDE.md`, and anything else not in `UNTRACKED_ROOT_FILE_ALLOWLIST` will be refused if introduced as untracked. That's by design — those tend to be edited via PRs that explicitly call out the doc/config change. If the refusal proves too aggressive in practice, extending the list is one line. The fail-closed semantics make this safe to iterate on.
- **`config.md`-driven allowlist deferred.** A future refinement could lift the hardcoded list into `projects/<slug>/config.md` so different projects with different layouts get different allowlists. D14 ships the hardcoded version per disqualifier "Making the allowlist configurable via config.md."
- **Two pre-existing integration tests adjusted, not added.** `commit: stages substrate paths` and `commit: excludes settings.local.json` both wrote files at the repo root (`someCode.ts`, `normal.ts`) that would now be refused. Updated to write under `lib/` (allowlisted). The test intent is unchanged; only the fixture-path is shifted to match the new policy.
- **Prelude commit (`c1751a0`) bundled on this branch dogfooded D13's `--reconcile` flag in production.** Running `autoload --reconcile` on the live agent-guilds project (manifest said `#39 (open)`, gh said merged) auto-flipped the row + appended a `pr-merged | #39` event row — visible inline in the briefing as `> Reconciled drift: PR #39 marked merged (was open).`. The reconcile commit + Phase 1.5 row branch flip to carryover-2 are housekeeping bundled with the D14 contract file in that single prelude commit; D14's actual behavioral diff is in commit `3bb8bac`.
