## Contract

- **Goal**: Augment `.claude/scripts/trout/autoload.ts` so PR state in the orientation briefing is verified against gh (`gh pr view --json state,mergedAt`) at run time rather than trusted from MANIFEST.md alone. Default mode surfaces drift in the briefing without mutating state. New `--reconcile` flag actively closes `(open) → merged` drift by invoking `autosave` to record a `pr-merged` event (which also flips the phase row's PR cell). `/ev-run` is updated to pass `--reconcile` so the user-facing orientation path auto-closes drift. This closes the manifest-vs-gh state-propagation gap that has required manual user intervention three times in Phase 1.5 (sessions 2026-05-08-a, 2026-05-08-b, and this session's PR #35 reconcile).

- **Acceptance criteria**:
  1. For each PR number referenced in the **Phases** table of MANIFEST.md (parsed from the existing PR column), autoload queries `gh pr view <N> --json state,mergedAt` and obtains a normalized state: `open`, `merged`, `closed`, or `unavailable`.
  2. gh queries run **in parallel** (Promise.all or equivalent) so a Phases table with 5 PRs doesn't multiply user wait time by 5.
  3. Default mode (no flag): briefing surfaces drift inline next to the PR cell — concrete shape `#N (open ⚠ merged on gh)`. The bare `(merged)` / `(open)` text is preserved when gh and manifest agree. **No MANIFEST mutation in default mode.**
  4. New `--reconcile` flag: for each `(open) → merged` drift detected (manifest says open, gh says merged), autoload invokes `node .claude/scripts/trout/autosave.ts <slug> --event=pr-merged --detail=#<N>` (which both records the event row AND flips the phase row's PR cell via `applyPRMergedToPhaseRow`). Reconciliation runs BEFORE the briefing renders, so the briefing reflects the post-reconcile state. Briefing also includes a one-line note for each reconciled PR (e.g. `> Reconciled drift: PR #33 marked merged (was open).`).
  5. `--reconcile` does **not** auto-fix the `(merged) → open` edge case (manifest claims merged but gh says open). Auto-flipping there would erase a recorded `pr-merged` event from history — destructive. Surface as a `⚠` warning inline in the briefing (same shape as default mode) regardless of `--reconcile`.
  6. `--reconcile` is a no-op when no drift exists (nothing to invoke autosave for).
  7. Graceful degradation when gh is unavailable (gh not installed, network error, not authenticated, exit non-zero for any reason — including a single failing `gh pr view` while other PRs succeed): briefing includes a single-line note `> _gh unavailable — PR states shown are manifest-only._` near the Phases table, and PR cells render as-is from manifest. `--reconcile` becomes an inert flag in that case (nothing to reconcile against). **autoload must not fail or hang on gh unavailability.**
  8. `/ev-run` skill body (`.claude/skills/ev-run/SKILL.md` step 1 "Orient") updated to invoke autoload with `--reconcile`. Without this wiring the new flag would be unused and the gap wouldn't actually close.
  9. Sibling `autoload.test.ts` adds coverage:
     - all-agree (no drift, no reconcile note)
     - `(open) → merged` drift, default mode (inline `⚠` indicator, no MANIFEST change)
     - `(open) → merged` drift, `--reconcile` (MANIFEST updated, event row recorded, reconcile note in briefing)
     - `(merged) → open` drift, `--reconcile` (warning surfaced, NOT auto-corrected — manifest untouched)
     - gh-unavailable fallback, default mode (advisory note, MANIFEST untouched)

## Execution

`autoload.ts` got three new responsibilities, all gated behind the existence of PRs in the Phases table (no PRs → no behavior change vs. pre-D13):

1. **gh-state querying.** New `queryGhPrState(prNumber)` wraps `child_process.spawn` in a Promise that resolves to `'open' | 'merged' | 'closed' | 'unavailable'`. Any failure path (gh not installed, network error, non-zero exit, JSON parse failure, ENOENT on spawn) collapses to `'unavailable'` — the Promise never rejects so `Promise.all` is safe.
2. **Drift-aware table renderer.** `renderPhasesTable(phases, ghStates)` rebuilds the markdown table from parsed `Phase[]` instead of rendering `manifest.phasesTable` verbatim. When manifest and gh disagree on a PR cell, it injects `(open ⚠ merged on gh)` (or symmetric) inline. When they agree, the original `(merged)` / `(open)` rendering is preserved.
3. **`--reconcile` flag.** New `parseArguments` returns `{ slug, reconcile }`. When set and gh isn't all-unavailable: for each `(open) → merged` drift, autoload invokes `node .claude/scripts/trout/autosave.ts <slug> --event=pr-merged --detail=#<N>` via `spawnSync`. autosave's existing `applyPRMergedToPhaseRow` does the cell flip; the event row gets recorded the same way every other manifest write does. After all reconcile invocations complete, autoload re-reads + re-parses the manifest so the rendered table reflects the post-reconcile state. Each successfully-reconciled PR adds a `> Reconciled drift: PR #N marked merged (was open).` block-quote line under the Phases table.

`(merged) → open` drift is **never** auto-corrected even with `--reconcile` — flipping there would erase a recorded `pr-merged` event from the events table. Surface-only via the `⚠` indicator.

`main()` was converted to async to support `await Promise.all(...)`. Top-level invocation became `main().catch(err => { stderr; exit 1 })` to preserve the prior fail-fast semantics.

`/ev-run` SKILL.md updated to invoke `autoload.ts <slug> --reconcile` (was: no flag) so the user-facing orientation path closes drift automatically. Without this wiring the `--reconcile` flag would be unused in practice.
