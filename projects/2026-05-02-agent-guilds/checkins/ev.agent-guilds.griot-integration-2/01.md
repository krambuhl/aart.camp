**Step 3 — A/B verification** documented in
`projects/2026-05-02-agent-guilds/codemod-exercises/5-griot-use-ab-test.md`.
Synthetic AP-001 entry + two `/tmp/ab-test-{a,b}/components/canvas.module.css`
fixtures with identical starting content. Ran `generator-css-codemod`
in parallel — Run A with no rollup, Run B with the synthetic rollup

**Step 4 — Outcome**:
- **Token choice**: identical (both picked
  `token("bg.inverted.default")`). Cold reasoning was strong enough
  to converge on this token's namespace, so the rollup wasn't the
  load-bearing reason for the right answer here. That's the
  "convergence" measurement caveat documented in the A/B doc.
- **Citation**: Run B emitted `Applied: AP-001` on its own line at
  the end of its narrative. Run A produced no citation. This is the
  **load-bearing falsifiable signal** — the rollup reached B's context
  and was visibly applied to its reasoning.
- **Framing**: Run A flagged the dark-mode inversion as "worth a
  human eye"; Run B treated it as the rollup's documented intentional
  behavior. Confidence + audit-trail effect is real.

## Notes for the PR

- **Phase 5 D2 closes the surface half of the substrate feedback
  loop**. D1 (PR #87) wired capture + classification + frequency-count
  + griot-compact routing. D2 (this PR) wires `/griot-use` injection
  of the `## Project antipatterns` section with top-10 curation, plus
  the synthetic A/B verification end-to-end. Phase 5 D3 (composability-
  proof loop `/a11y-review-file`) is the final unit before phase
  close.

- **A/B verification produced an honest outcome**: token choice
  converged (both runs picked `bg.inverted.default`) but Run B
  emitted `Applied: AP-001` while Run A did not. The citation is
  **the load-bearing falsifiable signal** for "rollup reached the
  generator's context"; convergence on token choice is a property
  of the specific antipattern + this generator's cold-reasoning, not
  a property the loop must produce. The A/B doc names this distinction
  explicitly under "Choice convergence as a measurement caveat" —
  per the contract-fit evaluator, "doesn't overclaim behavioral
  steering from a single-fixture exercise."

- **Top-N=10**: user picked the upper end of performance whiteboard's
  "5-10" range during AskUserQuestion. The cap is hardcoded in
  `ANTIPATTERN_TOP_N`; making it configurable is post-Phase-5 work.

- **Substrate findings worth elevating** to next `/griot-compact`:
  1. **parse-and-aggregate VERDICT-line strictness** — observed AGAIN
     (4th+ time across phases). The aggregator regex requires
     `^VERDICT:` at the start of a line; evaluators tend to embed
     the verdict in a paragraph. Coaching evaluators to put VERDICT
     on its own line works, but the parser could be more permissive
     (e.g. match VERDICT anywhere as long as it's preceded by
     whitespace or paragraph boundary). Worth a follow-up substrate
     unit. Per the dogfood-deferred rule, this finding is documented
     here but NOT captured as `--evaluator-finding=recurring` in
     this PR — it folds into a future substrate cleanup.
  2. **A/B convergence-on-correct-answer pattern**. When the rollup's
     preferred answer aligns with cold-reasoning's default, the A/B
     proves "rollup is present + applied" via the citation, but
     cannot prove "rollup caused the choice." A contrarian
     antipattern (where the rollup's preferred answer differs from
     cold-reasoning's default) would close that loop. Constructing
     one synthetically risks being artificial; the natural source is
     real captured findings from real project work, which is
     post-Phase-5 (dogfood deferred).
  3. **Citation-contract content always mentions both `L-NNN` and
     `AP-NNN` forms** — the LLM consumer picks the form per actual
     usage. Refactored from a dynamic builder to a single constant
     after the dynamic form broke the tier-separation invariant test.
     Constant form is cleaner; worth noting as the canonical shape.

  `evaluator-naming`) and the helper script (`derive-panel.ts`)
  predate this session. The A/B exercise spawned
  `generator-css-codemod` (also predates this session).
