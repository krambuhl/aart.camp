- **Design topic for the exercise**:
  **"Should aart.camp adopt a draft/published distinction for
  sketches?"**

  Rationale: this is a real question for the site (the
  portfolio framing in `.claude/CLAUDE.md` is "each sketch is a
  finished thing"; adding a draft state would shift that
  framing). It engages multiple engineers' genuine perspectives
  with natural variance: sketch-ideation has STRONG opinions
  (portfolio is finished thoughts, drafts are anti-pattern),
  design-systems has views on the URL/route shape, react-
  architect has views on the implementation surface,
  performance and a11y have minor stakes, skeptic has
  feature-creep and scope-of-moderation concerns. The topic is
  small enough to converge on in 2 rounds, big enough to
  produce real contradictions worth resolving.
