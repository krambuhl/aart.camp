## What happened

Picked up a project that had drifted: PR #13 had silently merged at 16:22 UTC the day before this session started, but the local MANIFEST never recorded the merge, so the autoload briefing reported #13 as "open" — and we acted on that fiction throughout the session. The substantive work (deliverable 6 part 1: extracting `/trout-pull-request` plumbing into `.claude/scripts/trout/pr-plumbing.ts` + 656-line sibling test) landed cleanly: 117/117 tests pass, evaluator approved on run 1 with 3 tool uses, and we ate our own dog food by using the new `commit` and `submit` verbs to ship the work itself. Two refinements to the verb design landed mid-session in response to user direction: (a) `commit` defaults to commit + push (`--no-push` opts out) so "push commits as they come" is the path of least resistance; (b) `submit` is atomic (gh action → autosave → git add MANIFEST → tracking commit → push) so a session can never end with a `pr-opened`/`pr-updated` autosave's MANIFEST update sitting uncommitted. The `submit` invocation hit two real bugs the unit tests didn't catch: a regex that excluded dots from repo names (broke on `aart.camp`) and the user typo'd `pr=#13` instead of `#16` in the phase-update arg, leaving the phase row stale until a follow-up `note` event corrected it. Net new PR (#16) carries the right substantive diff but also accidentally swept up 5 unrelated session-notes files from a prior session — a sign the `commit` verb's allowlist for untracked code outside `projects/` is too permissive.

  2. `pr-plumbing.ts` `commit` verb's untracked-files allowlist is too permissive for code outside `projects/`. Tighten to `.claude/`, `projects/`, and a curated set of code dirs (`components/`, `app/`, `sketches/`, `tokens/`, `utilities/`, `tests/`, `styles/`).

- **Lesson for the broader substrate**: post-merge state propagation is the soft underbelly. The merger of a PR is currently invisible to the local substrate unless someone runs `autosave --event=pr-merged`. Either the autoload should reconcile, or the merger should auto-record. Worth a project-level conversation.
