## Notes

- **PATH-override fixture gotcha (lesson learned during D9).** When tests override the spawned process's `env.PATH` to inject a stub binary (gh in this case), invoking `node` by name fails because PATH no longer includes node's nvm location. Symptom: 12 silent test failures with empty stderr. Fix: invoke node via `process.execPath` (absolute path) so name-lookup is bypassed. This is the second PATH-related fixture-fragility this phase — D8 had a similar one (`cpSync` imported from `node:child_process` instead of `node:fs`). Pattern: when a fixture builder overrides the test environment, audit what the shell needs from PATH/imports to actually run.
- **Soft-cap-as-binary lesson (D8 run 1 flagging).** Disqualifier-list line caps with "~N" prefixes get read as hard binary by the antagonist lens, even though "~" reads as "approximately" in normal English. The lens correctly resists soft-pedal — that's its job. Future contract phrasing: either a hard cap or an explicit "soft cap, document overruns" framing, not "~N" alone.
- **Compensatory-prose discipline (D9 SKILL compression).** When an LLM/CRUD split removes mechanical instruction prose from a SKILL.md, the prose tends to grow elsewhere ("here's what the script does") to fill the deletion. Initial D9 SKILL draft was 149 (+34 over the prior 115) before the actual mechanical content shrank. Future contracts should cap that compensatory prose explicitly, not just total length.
- **Two-step autosave pattern post-`submit` is still ugly.** Each new-PR submit produces TWO tracking commits — one for the `pr-opened` event row, one for the post-submit `phase-row PR field updated` note that records the gh-returned PR number. The session-b refinement #5 ("`submit <N>` placeholder substitution") would collapse both into one. Worth doing before the next round of new-PR submits if D10's cleanup-7 branch ends up wanting fewer tracking commits in its history.
