## Execution

Two TDD-ordered commits closing Phase 2:

- **TDD step 1** (commit `431fd83`): 7 test files referencing
  not-yet-existing modules. All 7 fail to import → true red state.
- **TDD step 2** (commit `1e16973`): 3 lib impls + 4 verb impls +
  dispatch wiring for checkin/session/retro/doctor + new
  `VERBLESS_NAMESPACES` set for doctor's `[<slug>]` shape.

Lib modules follow the unit-02 reader pattern — read JSON file,
parse, return typed value, throw `LoomError` on failure. checkin.ts
adds a recursive `scanCheckinsDir` walker since branch names with
slashes (e.g. `loom-cli/phase-1`) become nested subdirs under
`checkins/`.

Verb handlers wire the libs into dispatch with the same `emit()` +
`errToResult()` shape established in unit 03. Three notes worth
calling out:

1. **session corrections is loom-native, not a trout port.** Trout's
   `correction:` scanning regex-greps markdown checkin files. Loom's
   checkins are JSON with a structured `execution.corrections[]`
   field, so the verb just iterates `listCheckins()` and collects
   from each checkin's array. Output shape `[{text, checkin,
   branch}]` is what Phase 4's `loom-session` skill will consume.
2. **retro read filename derivation.** `--type=project` resolves to
   `retros/project.json`; `--type=session --phase=N --tier=M`
   resolves to `retros/phase-N-tier-M.json`. Filename convention is
   in code only; LOOM-CONVENTIONS.md should pick this up in a
   follow-up.
3. **doctor is verbless.** LOOM-CONVENTIONS.md documents `loom
   doctor [<slug>]` — namespace-as-verb. The dispatch surface
   previously assumed `<namespace> <verb> [rest]` for every
   namespace. Added a `VERBLESS_NAMESPACES = new Set(['doctor'])`
   to special-case the routing: when the namespace is verbless,
   the first verb-position arg is treated as a handler arg.
