# Checkin 01 — loom-cli/phase-3-lifecycle

**Created**: 2026-05-15 10:20
**Phase**: 3 — Lifecycle write API + lifecycle skills
**Unit**: Writer lib (manifest + events + config + checkin + session + retro)

## Contract

- **Goal**: Add write-side functions to each existing lib module so
  Phase 3's verbs (unit 02 onward) have a typed, structured-error
  writer surface to call. Writers are pure, idempotent where it
  makes sense, and never mix concerns with caller logic. Events get
  a special append-only `appendEvent` helper since events.jsonl is
  line-by-line.

- **Acceptance criteria**:
  1. `cli/lib/manifest.ts` exports `writeManifest(path, manifest)`.
     Writes pretty-printed JSON (2-space indent) for diff stability.
     Errors: `manifest-write-failed`.
  2. `cli/lib/events.ts` exports `appendEvent(path, event)`. Opens
     the events.jsonl file in append mode, writes the event as a
     single line (no trailing comma, trailing newline). Creates
     the file if missing. Errors: `events-write-failed`.
  3. `cli/lib/config.ts` exports `writeConfig(path, config)`. Same
     pretty-printed shape as writeManifest. Errors:
     `config-write-failed`.
  4. `cli/lib/checkin.ts` exports `writeCheckin(projectPath, checkin)`.
     Derives the target path from `checkin.branch` and `checkin.number`
     (zero-padded). Creates intermediate `checkins/<branch>/` dirs as
     needed (branch may contain slashes). Refuses to overwrite an
     existing file (checkins are immutable). Errors:
     `checkin-write-failed`, `checkin-already-exists`.
  5. `cli/lib/session.ts` exports `writeSession(projectPath, session)`.
     Derives target path `sessions/YYYY-MM-DD-<letter>.json`. Creates
     `sessions/` dir if absent. Refuses to overwrite. Errors:
     `session-write-failed`, `session-already-exists`.
  6. `cli/lib/retro.ts` exports `writeRetro(projectPath, retro)`.
     Derives target filename per the LOOM-CONVENTIONS.md scheme
     (`project.json` for type=project; `phase-<N>-tier-<M>.json` for
     type=session). Creates `retros/` dir if absent. Refuses to
     overwrite. Errors: `retro-write-failed`, `retro-already-exists`.
  7. Sibling tests extended: write + readback round-trip for each
     domain, immutability assertions for checkin/session/retro,
     append-only assertion for events (writing two events results
     in two lines, second event preserved).
  8. TDD discipline: tests-first commit, then impls commit.
  9. `npm run test` passes; biome lint clean.
  10. No new verbs (those land in unit 02+).

- **Rules applied**:
  - TDD: tests-first commit, then impls commit.
  - Stdlib only (`node:fs`).
  - Erasable TS.
  - Vitest sibling `.test.ts`.
  - LoomError shape from unit 02.
  - Pretty-print JSON (`null, 2`) for diff stability on writes.
  - No emojis.

- **Disqualifiers**:
  - Implements any CLI verbs (unit 02+).
  - Wraps writes in transactions or rollback (Phase 3's archive is
    explicitly non-atomic per PLAN).
  - Adds runtime JSON Schema validation libs.
  - Uses non-erasable TS.
  - Overwrites checkin/session/retro files (immutability invariant).

- **Inputs**:
  - `projects/LOOM-CONVENTIONS.md` — file shapes, event vocabulary
  - `.claude/cli/lib/types.ts` — schema types
  - `.claude/cli/lib/errors.ts` — LoomError pattern
  - Existing readers in each lib module as the structural mirror

## Execution

Two TDD-ordered commits adding the writer surface to the existing
six lib modules:

- **TDD step 1** (commit `29ed765`): 12 new tests across six test
  files. All write tests fail since the write* functions don't
  exist yet.
- **TDD step 2** (commit `f30e01e`): 6 write functions + small fix
  to the writeCheckin overwrite test (the fixture's branch field
  didn't match the test setup's branchDir, so the derived path
  didn't collide).

Writers all follow the same shape: stdlib `writeFileSync` /
`appendFileSync`, pretty-printed JSON with trailing newline for diff
stability, `LoomError` on failure with stable codes. The three
immutable file types (checkin / session / retro) guard with
`existsSync` before write and throw `*-already-exists`.

`retroFilename(retro)` is exported separately from `writeRetro` so
verbs and skills can derive the path without going through the
writer. Mirrors how `retroRead` (unit-04 phase 2) derives the
filename from `--type` flags.
