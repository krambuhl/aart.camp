**Unit**: Lifecycle bookends (project scaffold + project archive)

## Contract

- **Goal**: Add the two lifecycle bookends — `project scaffold`
  creates a loom-managed project from input files, `project
  archive` relocates it under `projects/archive/` after flipping
  its status. With these merged, loom owns the full lifecycle:
  birth, work (units 02 + Phase 2's read API), retirement.

- **Acceptance criteria**:
  1. `verbs/project.ts` adds `projectScaffold(rest, ctx)` and
     PROJECT_VERBS map grows `scaffold` key. Args: `<slug>
     --plan-file=<path> --config-file=<path>
     --manifest-init-file=<path>`.
  2. Slug handling: accepts date-less form (e.g. `loom-cli`),
     prepends today's date (`YYYY-MM-DD-<slug>`). Full slug form
     accepted verbatim. Invalid slugs (uppercase, special chars)
     throw `invalid-slug`.
  3. Manifest-init file shape: `{title, started, strategy,
     phases:[{number,name,status,...}]}`. Scaffold composes the
     full Manifest by merging this with the slug + `schema_version:
     1` + status:active + current_branch:null + latest_checkin:null.
  4. Scaffold writes: `manifest.json`, `config.json` (copied via
     readConfig+writeConfig), `events.jsonl` (with a single
     `project-initialized` event), `PLAN.md` (copied from
     --plan-file). Creates `checkins/` and `sessions/` empty
     dirs.
  5. Scaffold refuses to overwrite an existing project (slug
     collision in active or archive). Error: `project-exists`.
  6. Returns `{slug, path}` as JSON on success.
  7. `verbs/project.ts` adds `projectArchive(rest, ctx)` and
     PROJECT_VERBS map grows `archive` key. Args: `<slug>`.
  8. Archive sequence (non-atomic per PLAN.md):
     a. Resolve project, read manifest.
     b. Update `manifest.status = 'archived'`, write manifest back.
     c. Append `archived` event with `{destination}` detail.
     d. Move the project directory to `projects/archive/<slug>/`.
     If step (d) fails, status says archived but dir is still in
     active. `doctor` would surface the drift (future
     enhancement).
  9. Returns `{slug, destination}` as JSON on success.
  10. Sibling tests in `verbs/project.test.ts`: scaffold happy
      path (resulting tree shape + project-initialized event +
      idempotency check via second-call → project-exists), archive
      happy path (tree relocated + manifest updated + archived
      event), error cases (missing args, invalid slug, etc.).
  11. TDD discipline: tests-first commit, then impls commit.
  12. `npm run test` passes; biome lint clean.

## Execution

Two commits closing out the lifecycle write API:
- **TDD step 1** (`2e9705d`): 7 new tests in project.test.ts. All fail.
- **TDD step 2** (`36ea071`): projectScaffold + projectArchive + helpers.

projectScaffold composes the full project tree: directory, manifest
(merged from input + slug + defaults), config (round-tripped via
readConfig+writeConfig for validation), PLAN.md (file copy), an
initial project-initialized event, and empty checkins/+sessions/.
Refuses overwrite of either active or archive collisions.

projectArchive's sequence is deliberately non-atomic:
1. readManifest → flip status='archived' → writeManifest
2. appendEvent('archived', {destination})
3. mkdirSync(archive/) → renameSync(projectPath, destination)

If step 3 throws (e.g. filesystem permissions), the active-location
manifest already says status='archived' but the directory hasn't
moved. `doctor` (Phase 2 unit 04) will surface this drift in a
future enhancement.

Smoke verified end-to-end:
- scaffold creates project with date-prefixed slug
- list active shows it
- archive moves it; manifest flipped; archived event appended
- list active becomes empty; list --archived shows the entry

## Notes for the PR

- **Full lifecycle now works.** Smoke test confirmed end-to-end:
  scaffold → list → archive → list active/archived. Loops or skills
  can now create and retire loom projects through the CLI.
- **`--manifest-init-file=` flag added** beyond PLAN.md's literal
  list. Mirrors the trout-substrate's existing `plan-scaffold.ts`
  three-file pattern (plan + config + manifest-init). Without it,
  scaffold would have to parse PLAN.md prose for title / strategy /
  phases, which is fragile.
- **Non-atomic archive is intentional.** Manifest writes + event
  append happen before the directory rename. If the rename fails,
  the manifest at the active location says `status: 'archived'`
  but the directory hasn't moved. PLAN.md anticipates this as a
  drift state; `doctor` should surface it (Phase 2 enhancement,
  not in scope here).
- **UTC date prefix.** `normalizeSlug` uses UTC components when
  prepending today's date. Keeps slug generation deterministic
  across machines and matches the ISO timestamp convention loom
  uses everywhere else.
- `correction:` Continued `.claude/cli/**/*.ts` panel-composition
  gap and L-004 session-cache issue.
