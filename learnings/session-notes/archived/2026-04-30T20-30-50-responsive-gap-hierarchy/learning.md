When a `Stack`'s `direction` is responsive (e.g., `{ xs: 'vertical', sm: 'horizontal' }`), its `gap` value is interpreted on different *axes* across breakpoints. A fixed gap that reads as comfortable horizontal breathing room becomes a vertical gap on the smaller breakpoint, where it can compete with the outer container's gap on the same axis. If the inner gap is larger than the outer container's gap on `xs`, items inside each row will visually pair with the wrong neighbor (e.g., a date sitting closer to the next title than to its own).

When a Stack changes direction across breakpoints, make `gap` responsive too. Pick the inner gap on `xs` to be visibly smaller than the outer container's `xs` gap so nested pairs read as a unit. Keep the larger `gap` value on `sm+` where it lives on the horizontal axis and doesn't compete with vertical row spacing.
