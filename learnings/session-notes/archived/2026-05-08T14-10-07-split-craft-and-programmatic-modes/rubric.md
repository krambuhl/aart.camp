attempt to modify this file is a hard violation._

- Output identifies that the skill/tool being migrated has more than one distinct usage mode (e.g., interactive in-conversation use AND programmatic/CLI use) before proposing a migration shape
- Output does not recommend fully deleting the interactive/craft-work workflow; the interactive mode is preserved as a skill (or equivalent user-facing primitive) rather than collapsed into the script/CLI
- Output proposes splitting the two modes across different primitives (skill for interactive, script for programmatic) rather than housing both modes in a single tool
